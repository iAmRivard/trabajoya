Objetivo general
----------------
Construir TrabajoYa, un flujo de agente de voz para ciudadanos comunes en El Salvador.
La idea es que el usuario pueda cargar un CV opcional o hablar con un agente de voz.
El agente toma datos tipo CV, completa lo que falte con preguntas, resume el perfil,
pide confirmacion y guarda el perfil estructurado para luego recomendar empleos,
cursos y entrevistas simuladas.

Flujo elegido
-------------
Opcion B:

CV opcional -> agente de voz confirma/completa datos -> n8n online -> PostgreSQL/Supabase

Por ahora se prioriza la fase de captura de perfil laboral.
Recomendaciones, cursos y entrevistas se agregan despues.

Marca y tono
------------
Marca: TrabajoYa
Pais: El Salvador
Tipo de usuario: usuario comun ciudadano
Tono del agente: neutro latino

Reglas importantes del agente
-----------------------------
- No prometer empleo.
- No pedir DUI ni documentos sensibles.
- No preguntar religion, genero, apariencia, orientacion sexual, salud,
  afiliacion politica ni datos financieros.
- Para edad usar rangos, por ejemplo 18-24, 25-34, 35-44, 45+.
- Si hay experiencia informal, convertirla en habilidades laborales.
- Hacer una pregunta a la vez.
- Si el usuario subio CV, confirmar lo encontrado y preguntar solo lo que falte.
- Antes de guardar, resumir el perfil y pedir confirmacion.

Workflow minimo de n8n
----------------------
1. Webhook node:
   POST /webhook/trabajoya/create-candidate-profile

2. Code node:
   Validar que exista profile.personal.full_name.

3. Postgres/Supabase node:
   Insertar en candidate_profiles:
   full_name = profile.personal.full_name
   email = profile.personal.email
   phone = profile.personal.phone
   municipality = profile.personal.municipality
   department = profile.personal.department
   source = source
   profile = profile
   raw_cv_text = raw_cv_text
   conversation_summary = conversation_summary

4. Respond to Webhook:
   Devolver:
   {
     "ok": true,
     "candidate_id": "uuid-generado-en-postgres",
     "message": "Perfil guardado. El siguiente paso es revisar recomendaciones de empleo."
   }

Decision tecnica minima
-----------------------

n8n + Postgres + ElevenLabs webhook tool + cliente local.

Sirve para leer perfiles guardados desde Postgres usando el tunel SSH local
`localhost:15432`, sin exponer Postgres publicamente.

El backend local se podra ampliar para:
- Extraer CV en PDF con mas control.
- Validar payloads con reglas mas estrictas.
- Manejar autenticacion de usuarios.
- Servir dashboard/frontend.
- Calcular recomendaciones con logica propia.
